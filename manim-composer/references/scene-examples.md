# Scene Examples

Use these as planning patterns. They are not implementation templates, but they show the kind of breakdown the composer should produce.

## Example 1: Explaining A Formula

1. Title card that states the question.
2. Introduce the core equation.
3. Highlight the part that changes the meaning.
4. Derive the intuition with a simple diagram.
5. Close with a short summary of the result.

## Example 2: Visualizing A Process

1. Show the initial state.
2. Introduce the moving part.
3. Let the viewer observe the transition.
4. Annotate the key relationship.
5. Compare before and after.

## Example 3: Graph-Based Explanation

1. Present the axes and the quantity being measured.
2. Add the function or curve.
3. Mark a key point or interval.
4. Show the slope, area, or change of interest.
5. Return to the whole picture and state the takeaway.

## Example 4: Camera-Driven Exposition

1. Start wide to show the whole structure.
2. Zoom into the part that matters.
3. Keep the central object fixed while details update.
4. Pull back to re-establish context.
5. End with the full map of the idea.

## What The Plan Should Include

- scene names
- beat order
- objects introduced in each scene
- what changes over time
- where pauses belong
- what must be reused across scenes
